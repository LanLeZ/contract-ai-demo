# 功能修改测试方案

## 测试范围

本次测试涵盖以下两个功能的修改：
1. 合同列表展示功能修改
2. 文件上传功能修改（支持TXT和PNG，合同详情页面显示文件内容）

## 二、文件上传功能测试

### 2.1 TXT文件上传测试

**前置条件：**
- 准备一个测试用的TXT文件（包含中文和英文内容）
- 文件大小小于50MB

**测试步骤：**
1. 登录系统
2. 进入合同列表页面
3. 点击上传区域或拖拽TXT文件
4. 等待上传完成
5. 点击上传的合同查看详情

**预期结果：**
- 文件上传成功，显示成功提示
- 合同列表中出现该TXT文件
- 文件类型标签显示为"TXT"（橙色）
- 合同详情页面可以正常显示文件内容

**测试用例：**
- [ ] 上传UTF-8编码的TXT文件
- [ ] 上传GBK编码的TXT文件
- [ ] 上传包含特殊字符的TXT文件
- [ ] 上传空TXT文件（应提示错误）
- [ ] 上传超过50MB的TXT文件（应提示错误）

### 2.2 PNG文件上传测试

**前置条件：**
- 准备一个测试用的PNG图片文件（包含文字内容，用于OCR识别）
- 文件大小小于50MB
- 确保已安装OCR库（paddleocr或pytesseract）

**测试步骤：**
1. 登录系统
2. 进入合同列表页面
3. 点击上传区域或拖拽PNG文件
4. 等待上传和OCR处理完成
5. 点击上传的合同查看详情

**预期结果：**
- 文件上传成功，显示成功提示
- OCR识别成功，提取图片中的文字
- 合同列表中出现该PNG文件
- 文件类型标签显示为"PNG"（紫色）
- 合同详情页面显示OCR识别出的文本内容

**测试用例：**
- [ ] 上传包含中文文字的PNG图片
- [ ] 上传包含英文文字的PNG图片
- [ ] 上传包含中英文混合的PNG图片
- [ ] 上传纯图片无文字的PNG文件（应能处理，但内容为空）
- [ ] 上传超过50MB的PNG文件（应提示错误）
- [ ] 验证OCR识别准确性（对比原图文字）

**注意事项：**
- 如果未安装OCR库，上传PNG文件会失败并提示安装
- 推荐使用PaddleOCR（pip install paddleocr）
- 备选方案：pytesseract + pillow（pip install pytesseract pillow）

### 2.3 合同详情页面测试（TAB1）

**测试步骤：**
1. 上传任意支持的文件类型（PDF/DOCX/MD/TXT/PNG）
2. 点击合同列表中的合同项
3. 查看右侧合同详情页面

**预期结果：**
- 显示合同详情卡片，标题为文件名
- 有"返回"按钮可以返回上传区域
- 显示"合同详情"标签页（TAB1）
- TAB1包含以下内容：
  - 文件ID（代码格式显示）
  - 文件类型（带颜色的标签）
  - 文件大小（MB格式，如果有）
  - 分块数量（如果有）
  - 上传时间（完整日期时间）
  - 文件内容预览区域（显示解析后的文本内容）

**测试用例：**
- [ ] 验证TAB1显示所有字段信息
- [ ] 验证文件内容预览区域正确显示文本
- [ ] 验证长文本内容可以滚动查看
- [ ] 验证文本格式保持（换行、空格等）
- [ ] 验证空内容显示"暂无内容"
- [ ] 验证返回按钮功能正常
- [ ] 验证不同文件类型的详情显示正确

### 2.4 文件类型支持测试

**测试步骤：**
1. 分别上传PDF、DOCX、MD、TXT、PNG文件
2. 查看文件类型标签显示

**测试用例：**

## 已知问题和限制

1. **PNG OCR依赖：**
   - 需要安装OCR库才能支持PNG文件
   - 首次使用PaddleOCR会下载模型，需要网络连接

2. **大文件处理：**
   - 超大PNG文件的OCR处理可能较慢
   - 建议限制PNG文件大小或提供进度提示

3. **编码检测：**
   - TXT文件编码检测基于尝试机制，可能无法100%准确
