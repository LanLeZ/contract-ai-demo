# 为什么Embedding能判断语义相似度？

## 什么是Embedding？

**Embedding（嵌入）**是将文本转换为高维向量的过程。这些向量是数字数组，能够捕捉文本的语义信息。

例如：
- 文本："中华人民共和国是社会主义国家"
- Embedding向量：`[0.123, -0.456, 0.789, ..., 0.234]` (1024维)

## 为什么向量能表示语义？

### 1. **神经网络学习语义关系**

Embedding模型（如通义千问的`text-embedding-v3`）是通过**大规模文本数据训练**的神经网络：

- **训练数据**：包含数亿甚至数十亿的文本对（相似文本、相关文本、问答对等）
- **学习目标**：让语义相似的文本在向量空间中**距离更近**
- **训练过程**：模型学习将文本中的**语义特征**编码到向量的每个维度中

### 2. **向量空间的语义结构**

训练后的模型会在高维向量空间中形成**语义结构**：

```
向量空间中的语义分布：

[相似的文本] ←→ 距离近（高相似度）
[相关的文本] ←→ 距离中等
[不相关的文本] ←→ 距离远（低相似度）
```

### 3. **具体例子**

假设我们有以下文本：

```python
文本1 = "中华人民共和国是工人阶级领导的、以工农联盟为基础的人民民主专政的社会主义国家。"
文本2 = "中华人民共和国的一切权力属于人民。"
文本3 = "今天天气很好，适合出去散步。"
```

经过Embedding后：
- **文本1和文本2**（都是宪法条文，语义相关）
  - 向量：`vec1 = [0.1, 0.2, ..., 0.3]`
  - 向量：`vec2 = [0.12, 0.19, ..., 0.28]`
  - **相似度高**（向量接近）

- **文本1和文本3**（宪法 vs 日常对话，语义不相关）
  - 向量：`vec1 = [0.1, 0.2, ..., 0.3]`
  - 向量：`vec3 = [-0.5, 0.8, ..., -0.2]`
  - **相似度低**（向量距离远）

## 如何计算相似度？

### 余弦相似度（Cosine Similarity）

最常用的方法是计算两个向量的**余弦相似度**：

```python
import numpy as np

def cosine_similarity(vec1, vec2):
    """
    计算两个向量的余弦相似度
    返回值范围: [-1, 1]
    - 1: 完全相同
    - 0: 正交（无关）
    - -1: 完全相反
    """
    dot_product = np.dot(vec1, vec2)
    norm1 = np.linalg.norm(vec1)
    norm2 = np.linalg.norm(vec2)
    return dot_product / (norm1 * norm2)
```

**为什么用余弦相似度？**
- 关注向量的**方向**而非大小
- 即使文本长度不同，也能比较语义相似性
- 结果在[-1, 1]范围内，易于理解

## Embedding模型的训练原理

### 1. **对比学习（Contrastive Learning）**

模型通过对比学习来理解语义：

```
训练样本：
- 正样本对：(问题, 正确答案) → 相似度高
- 负样本对：(问题, 错误答案) → 相似度低

模型学习：
- 让正样本对的向量更接近
- 让负样本对的向量更远离
```

### 2. **上下文理解**

Transformer架构（Embedding模型的基础）能够：
- 理解**词汇的上下文**（同一个词在不同语境中的含义）
- 捕捉**长距离依赖**（句子中远距离词汇的关系）
- 学习**语法和语义结构**

### 3. **多任务学习**

模型同时学习多个任务：
- 文本分类
- 问答匹配
- 文本相似度
- 语义检索

这些任务共同帮助模型理解文本的深层语义。

## 实际应用：向量检索

在你的法律检索系统中：

```python
# 1. 用户查询
query = "劳动合同的订立原则"

# 2. 转换为向量
query_vector = embedder.embed_query(query)

# 3. 在向量库中搜索最相似的文档
results = vector_store.search(query_vector, top_k=5)

# 4. 返回语义最相关的法律条文
```

**为什么有效？**
- 即使用户查询和法律条文**用词不同**，只要**语义相关**，向量就会相似
- 例如："劳动合同订立" 和 "订立劳动合同应当遵循合法、公平..." 虽然用词顺序不同，但语义相同

## 总结

1. **Embedding是语义的数学表示**：将文本的语义信息编码为高维向量
2. **相似文本 → 相似向量**：通过大规模训练，模型学会了这个映射关系
3. **向量距离 = 语义距离**：在向量空间中，距离近的文本语义相似
4. **余弦相似度衡量语义**：通过计算向量夹角来判断语义相似程度

这就是为什么只用Embedding就能判断语义相似度的原因！
