# 文本切分质量验证脚本使用说明

## 概述

`verify_text_splitting.py` 是一个用于验证文档文本切分质量的工具。它可以帮助你检查合同文档的分词是否合理，确保语义完整性，避免将语义单元（句子、段落）拆分。

## 功能特性

### 1. 基本统计
- 文档总长度和行数
- Chunk数量
- Chunk大小分布（最小值、最大值、平均值、中位数）
- Chunk大小分布直方图

### 2. 重叠统计
- 实际重叠字符数
- 与预期重叠的对比
- 重叠范围分析

### 3. 语义完整性检查
- **句子完整性**：检查chunk是否在句子中间截断
- **段落完整性**：检查chunk是否跨越段落边界
- **问题定位**：精确指出哪些chunk存在语义拆分问题

### 4. 质量评估
- 自动计算质量分数（0-100分）
- 质量等级评定（优秀/良好/一般/需要改进）
- 提供改进建议

## 使用方法

### 基本用法

```bash
# 验证文档（使用默认参数：chunk_size=500, overlap=50）
python scripts/verify_text_splitting.py --file uploads/2/internship1.docx
```

### 自定义参数

```bash
# 指定chunk大小和重叠
python scripts/verify_text_splitting.py \
    --file uploads/2/internship1.docx \
    --chunk-size 500 \
    --overlap 50

# 指定来源名称
python scripts/verify_text_splitting.py \
    --file uploads/2/internship1.docx \
    --source-name "实习协议"
```

### 参数说明

- `--file` (必需): 要验证的文档路径
- `--chunk-size` (可选): Chunk大小，默认500
- `--overlap` (可选): Chunk重叠大小，默认50
- `--source-name` (可选): 来源名称，默认使用文件名

## 常见问题

### Q1: 为什么实际重叠小于预期重叠？

**A**: 这可能是因为：
1. 切分器优先在句子/段落边界切分，而不是严格按照重叠大小
2. 文档结构导致某些位置无法实现完全重叠
3. 如果实际重叠过小（<10字符），可能需要调整切分策略

### Q2: 句子完整性多少算合理？

**A**: 
- **>95%**: 优秀，切分策略很好
- **90-95%**: 良好，有少量问题但可接受
- **80-90%**: 一般，需要优化
- **<80%**: 较差，需要改进切分策略

### Q3: 如何改进切分质量？

**建议**：
1. **调整chunk_size**: 如果句子经常被截断，可以适当增大chunk_size
2. **调整overlap**: 增加overlap可以改善上下文连续性
3. **优化切分器**: 修改`LawTextSplitter`的separators，优先在句子结束符（。！？）处切分
4. **使用语义切分**: 考虑使用更智能的切分方法，如基于语义的切分

### Q4: Chunk大小分布不均匀怎么办？

**A**: 
- 如果大部分chunks都很小（<300字符），说明切分过细，可以增大chunk_size
- 如果大部分chunks都很大（>600字符），可能超过模型限制，需要减小chunk_size
- 理想情况下，chunks应该集中在目标大小附近（如400-500字符）

## 技术细节

### 句子完整性检查

脚本通过以下方式检查句子完整性：
1. 识别中文句子结束符：`。！？；`
2. 识别英文句子结束符：`. ! ? ;`
3. 检查chunk边界是否在句子结束符处
4. 检查chunk是否在段落边界切分

### 重叠计算

脚本通过比较相邻chunk的末尾和开头来计算实际重叠：
- 从chunk1的末尾和chunk2的开头查找相同部分
- 计算最大匹配字符数作为重叠大小
